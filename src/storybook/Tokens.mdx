# Design tokens

Een probleem met bestaande NLDS-implementaties is de wildgroei aan tokens. Design-tokens zijn echter handig om een huisstijl in te defineren.
Aangezien elk component in MOX-nlds opgebouwd is uit Atoms, is er voor gekozen om de Atoms aanpasbaar te maken met design-tokens.

## Onderverdeling tokens

In andere NLDS-implementaties zijn er 3 levels van tokens:

- Brand tokens: de huisstijl, basis-instellingen zoals een **default font-size**
- Common tokens: semantische betekenis bij tokens, zoals de **default font-size voor buttons**
- Component tokens: een grote lijst met component-specifieke tokens, zoals **de font-size op de heading-button van accordions**

In MOX-nlds gebruiken we de laatste laag expliciet _niet_. Twee redenen:

- Een hoop tokens geven de mogelijkheid om van de Rijkshuisstijl af te wijken, bv door een ander font in een accordion te gebruiken. Dit is niet gewenst.
- Tegelijkertijd mis je vaak de tokens die je wel nodig hebt voor layout-invloed (bv een optie voor CSS-waardes voor `display`, `order`, `flex`, etc).

## Opzet tokens in json

```json
{
  "brand/color": {
    "mox": {
      "color": {
        "lintblauw": {
          "50": {
            "$type": "color",
            "$value": "#dce3ea",
            "$description": "Lintblauw 15%"
          },
          "100": {
            "$type": "color",
            "$value": "#b8c6d5",
            "$description": "Lintblauw 30%"
          },
          ...
```

Alles zit in een folder om dingen makkelijk terug te vinden in Figma - Tokens Studio. De folders zijn geprefixed met `brand` of `common` (we hebben zoals gezegd geen `component` tokens)

Dit wordt omgezet in CSS-variables:

```css
:root {
  --mox-color-lintblauw-50: #dce3ea; /** Lintblauw 15% */
  --mox-color-lintblauw-100: #b8c6d5; /** Lintblauw 30% */
  --mox-color-lintblauw-200: #95a9c0; /** Lintblauw 45% */
  ...
}
```
