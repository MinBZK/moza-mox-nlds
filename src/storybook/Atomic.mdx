# Atomic system

In een atomisch systeem zijn alle onderdelen onder de verdelen in een hierarchische structuur, in mate van complexiteit:

<img src={AtomicDesignSystem} alt="Opbouw van een atomic design system" />

Kort gezegd betekent dit:

- Atoms: dit zijn de kleinste bouwblokjes mogelijk
- Molecules: deze zijn opgebouwd uit atoms
- Organisms: opgebouwd uit molecules (en evt. atoms)

Molecules en Organisms zijn een onderscheid waar we voor MOX-nlds niet direct toegevoegde waarde zien. We voegen deze samen tot 1 groep, waardoor we onze onderverdeling krijgen:

- **Atoms**: kleinste bouwblokjes, leunen op design-tokens
- **Components**: complexer, opgebouwd uit atoms, gebruiken geen design-tokens
  ()

Voor een Design System dat potentieel door een brede gebruikersgroep gebruikt moet worden is dit handig.

- Sommige mensen willen snel een standaard pagina op kunnen zetten, en kunnen simpelweg een template gebruiken, of het via standaard componenten opzetten.
- Anderen willen juist controle hebben en componenten kunnen aanpassen op detail-niveau; zij hebben controle nodig en kunnen via atoms snel hun eigen custom component opzetten als de default niet volstaat.
- ... en iedereen die daar tussen valt

Met een atomic design system kan je de uitgebreide `organisms` of zelfs `templates` gebruiken om snel een pagina op te zetten.
Maar je kan ook `organisms` of `molecules` afpellen tot de `atoms` voor volledige aanpasbaarheid.
Iedereen kan via `design tokens` een eigen huisstijl toepassen, binnen de grenzen van de Rijkshuisstijl.
Het Mox Design system is een poging om aan de wensen van een brede gebruikersgroep te voldoen.

### lv. 0 - Tokens (CSS)

In de basis kan je via tokens je brandp-colors/spacing opties instellen voor een unieke huisstijl (of je gebruikt een standaard set tokens)

### lv. 1 - Atoms (CSS / React components)

**Atoms** zijn de kleinste bouwblokjes van het design-systeem.

Er zijn zowel Atoms voor CSS als voor React. De React componenten gebruiken de CSS classes.

#### Beloftes

- Er zullen altijd uitzonderingen zijn, maar in principe kan je met **atoms** bijna alle grotere componenten opbouwen
- De styling wordt altijd gedaan via CSS (zie de `Mox-css atoms`)
- Uitgebreide aanpasbaarheid qua styling via `design tokens`
- Consistente API tussen alle `atom` componenten:
  - `as`, `ref`, `className` en `styling` props
- Nooit omringende spacing zoals `padding` of `margin` (gebruik altijd layout-componenten zoals `stack` of `box` om de spaces tussen elementen te bepalen)
- Atoms zullen bijna nooit breaking changes hebben bij updates

### lv. 2 - Components (React components)

**Components** zijn opgebouwd uit **atoms** en zijn daardoor complexer. Ook kan er interactiviteit in voorkomen (bv bij tabs).

#### Beloftes

- _Geen_ `design tokens`
- Aanpasbaarheid door de API van het component zelf
- Docs over hoe het opgebouwd is (welke **atoms**), zodat je zelf makkelijk het kan nabouwen met **atoms** voor volledige aanpasbaarheid
- Anders dan **atoms** kunnen hier sneller breaking changes in voorkomen
- Nooit omringende spacing zoals `padding`, `margin` of (text-box ruimte)[?path=/docs/moxcss-text-box-trim--docs].
