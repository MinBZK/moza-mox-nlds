    # CSS Layers

Om goede controle te houden over specificity van CSS maken we gebruik van een set layers. De onderste layer heeft de laagste specificity, dus alles wat in een hogere layer gedefineerd staat, zal die styling overschrijven.

Als je zelf CSS styling opzet waarbij je de styling van MOX-nlds wilt overschrijven, neem je dus de laatste layer (`overrides`).

Soms komt het voor dat er al bestaande styling is in een project, wat je (langzaam) wilt overschrijven met MOX-nlds. In dat geval neem je juist de `legacy` layer. Als je een grote berg CSS code hebt, is het het makkelijkst om dit in het geheel te wrappen in een `legacy`layer.

```css
/* layer syntax */

@layer legacy {
  p {
    font-size: 15px; /* Deze styling blijft werken, totdat je een Paragraph-element gebruikt met MOX-nlds: dan zal die styling dit overschrijven. */
  }

  a {
    /* ... etc */
  }
}
```

## De layers (onderste layer eerst)

### 1. reset

De laagste layer, CSS-reset rules voor consistente browser-experience. Dit is dus nog lager dan enige `legacy` styling

### 2. legacy

Wordt niet door MOX-nlds gebruikt, maar enkel ingesteld voor oude CSS-code in je code-base die je wilt uitfaseren door stapje voor stapje over te gaan op MOX-nlds.

### 3. tokens

De CSS-variables die gebruikt worden door de **Atoms**.

### 4. base

Wat standaard CSS instellingen voor het MOX-nlds, zoals focus-styles

### 5. atoms &

### 6. molecules &

### 7. organisms

Styling rules voor de atoms, molecules en organisms (simpele bouwblokjes naar meer complexe componenten). In CSS zal veruit het meeste gedefinieerd zijn in de `atoms` layer. Molecules en Organisms zijn meer dingen die gebruikt worden voor React-componenten, maar heel soms is daarbij nog een klein beetje styling nodig - daarvoor gebruiken we dan de `molecules` of `organisms` layers.

### 8. props

Styling-properties (zie [properties](http://localhost:6006/?path=/story/moxcss-atoms-properties--all-generated-css)); deze zullen altijd boven de 'standaard' atom styling komen, omdat een gebruiker deze expliciet heeft bepaald (bv een `gap` van size `md`) en dus standaard css moeten overschrijven.

### 9. templates &

### 10. pages

Op dit vlak van complexiteit (deze zijn opgebouwd uit atoms, molecules en organisms van het MOX-nlds) kan je deze layers gebruiken om styling te overschrijven.

### 11. overrides

Voor als je zeker wilt zijn dat bepaalde styling de hoogste specificity krijgt. Maak je hier vaker gebruik van, dan is het waarschijnlijk een goed idee om even bij MOX-nlds aan te kloppen om te zien of jouw styling niet beter meegenomen kan worden in de standaarden van MOX-nlds.

## Definitie

Deze layers zijn zo gedefinieerd in de centrale CSS-file:

```css
@layer reset, legacy, tokens, base, atoms, molecules, organisms, props, templates, pages, overrides;
```
