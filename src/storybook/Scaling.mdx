# Scaling (afstanden, groottes)

Alles in MOX-nlds is gebaseerd op één maatvoering, namelijk de _fluid, incremental type-scale_ (responsive en in logische, voorspelbare stappen).
We hebben een vaste waarde voor 'normale tekst-grootte' gedefineerd in de volgende variabelen:

Verder is er de factor (`brand.globals.mox.font-scale.const.scale`) waaruit elke volgende stap vanuit deze basis berekend wordt. Deze is als default ingesteld op `1.2` (d.w.z: elke stap groter in font-size is de vorige size \* 1.2)

De basis-grootte noemen we `md` (medium) en heeft een factor `0`. Kleinere formaten worden berekend met de scale met factor `-1`, `-2`, etc. Grotere met `1`, `2`, etc. Één maat kleiner dan de minimale `md` (dus `sm` voor small) wordt berekend als volgt:

```
font-size-sm-min: font-size-md-min * scale ^ factor
```

In principe zijn dit vaste waardes, maar ze kunnen custom aangepast worden als token, waarna de nieuwe CSS-variables gegenereerd moeten worden via het `build:tokens` script.

## Clamping

De font-groottes (en alle andere spacing) lopen dus via deze scale omhoog of omlaag. Via een `min` en `max` versie zorgen we via het CSS-statement `clamp` dat alle spacing meeschaalt met je schermgrootte.

## Gebruik in Figma

Design in Figma met deze tokens is zo ingericht zodat je makkelijk het verschil kan zien tussen de verschillende schermgroottes. Door gebruik te maken van `themes` kan je schakelen tussen verschillende scherm-groottes, waardoor je direct de spacing in je design ziet veranderen.
